Server-Side Request Forgery (SSRF)

The vulnerable fetcher below asks the Flask server to make an HTTP request to any URL you give it - with no validation. Try http://127.0.0.1:5000/internal/secret-report, an endpoint that returns 403 to a direct browser request but succeeds when the server fetches it on your behalf.

Warning: this simulates reaching an internal-only service through the app server, the same shape of bug used against real cloud metadata endpoints. Only target this local instance.
