Two-stage design: a deep CNN turns each face into a feature vector, then a maximum-margin classifier draws the real-vs-morph boundary. Separating the two stages is what keeps a small dataset from over-fitting.

01 · System Architecture

The end-to-end pipeline

Why an SDLC view matters here. The two most expensive phases (morph generation ~3–6 h, feature extraction ~1.5–2 h) are idempotent and cached. Re-running after a crash skips completed work, so the lifecycle is fault-tolerant by construction — a real engineering requirement on free-tier Colab.
03 · Cell-by-cell

What each cell does

Expand any phase to see the key cell(s), the exact operation, and the output it produces. Grouped by lifecycle part.

feature / CNN generation / decision evaluation setup / IO
P1 Configuration & install ›

Defines the control switches (DEMO, DO_FINETUNE, CHECKPOINT_TO_DRIVE), the per-split image counts, global constants (INPUT_SIZE=528, MORPH_ALPHA=0.5, seeds), and installs libraries. Mounts Google Drive once for checkpointing.

Output: reproducible environment; a printed target of 24,000 real + 21,000 morph = 45,000 images.

P2 Download data + identity file ›

Downloads the 202,599 CelebA faces via kagglehub. Then obtains identity_CelebA.txt automatically in three fallbacks: search the download, then gdown from the official mirror, then a pseudo-identity fallback so the pipeline never blocks on a manual upload.

Why the identity file? It maps each image to a person so the same identity never appears in two splits — the guarantee that makes the accuracy honest.

P3 Identity-disjoint split + realistic morphs ›

Split: whole identities are shuffled and assigned to train/val/test so no person leaks across splits (verified by an assertion, including morph source identities).

Morph generation for each pair of different people in the same split:

# realistic seamless morph (not a naive frame blend)
p1,p2 = get_landmarks(img1), get_landmarks(img2)   # 468 pts
f2     = _color_transfer(img2, img1)          # match tone (LAB)
avg    = (1-α)*p1 + α*p2                     # average shape
tris   = Delaunay(avg).simplices             # triangulation
w1,w2  = _warp_to(f1,...), _warp_to(f2,...)  # affine per triangle
morph  = (1-α)*w1 + α*w2                     # blend inner face
out    = seamlessClone(morph, w1, hull, ...)  # Poisson, no seams

Automatic review: a morph is kept only if a face is still detectable and Laplacian sharpness ≥ threshold. Output: labelled dataset + labels.csv.

P4 Preprocess (528 + CLAHE) + augment ›

Resizes to 528×528 (B6's required input) and applies CLAHE on the LAB L-channel to lift local contrast without amplifying noise. Identical steps for real and morph so the model can't cheat on a preprocessing artifact. Augmentation (flip, brightness, blur, noise, JPEG re-compression) is defined for training robustness.

Output: a uniform 528px processed set, cached to Drive once.

P5 EfficientNet-B6 feature extraction ›

Loads B6 with ImageNet weights, include_top=False, adds Global Average Pooling → a 2304-d vector per image. Runs frozen (Phase 1); optional fine-tuning (Phase 2) unfreezes only block7+top at lr 1e-5. Features are cached to .npz locally and on Drive.

base = EfficientNetB6(include_top=False, weights="imagenet")
gap  = GlobalAveragePooling2D()(base.output)   # F ∈ R^2304
P6 SVM training ›

Standardizes features, grid-searches C on the validation split, fits the final SVM, and verifies the decision function by hand — recomputing sign(wᵀF+b) and asserting it equals predict().

Output: weight vector w ∈ ℝ2304 and bias b.

P7–P9 Results, threshold, architecture compare ›

P7 draws the confusion matrix + ROC/AUC and prints accuracy, precision, recall, F1. P8 sweeps the decision threshold on validation and picks the F1-optimal point. P9 re-runs the same pipeline with B0 and B5 backbones to reproduce the paper's Table 2 comparison.

P10 Real-time prediction ›

Uploads an image, detects every face (MediaPipe), and for each face runs preprocess → B6 → SVM → MORPH / REAL with a sigmoid confidence, plus a per-stage latency measurement. Supports single and multi-face images.

P11–P12 Save + improvement comparison ›

P11 saves the extractor, SVM, scaler and threshold to Drive. P12 applies the improvements (calibration, cost-sensitive & EER thresholds, ISO/IEC 30107-3 metrics) and prints a before/after table and DET plot.

04 · Algorithms & Mathematics

Every algorithm — why, and how

4.1  Morph generation — geometric warp + Poisson blend

Why: a detector is only as good as its attacks. A naive full-frame blend leaves tells (double hairline, colour seams) the model learns instead of real morphing. The seamless pipeline forces it to learn genuine artifacts.

How: for two faces with landmark sets p1, p2:

Average face shape (α = ½) p̄ = (1 − α)p1 + α p2
Reinhard colour transfer in LAB (per channel c) c' = (c − μsrc) · σref / σsrc + μref
Per-triangle affine warp to the average shape [x' y' 1]⊤ = M [x y 1]⊤,   M = Adst Asrc−1
Texture blend, then Poisson seamless clone over region Ω Imorph = (1−α)W1 + αW2;   solve ∇2f = ∇2Imorph in Ω, f|∂Ω = background

Delaunay triangulation maximises the minimum angle, giving stable (non-sliver) triangles so the affine warp stays smooth.

4.2  CLAHE — contrast without noise

Why: morph seams are subtle local contrast changes; global equalisation would blow up noise. CLAHE equalises each tile but clips the histogram first, capping amplification.

Clip-limited histogram equalisation (per tile) s = (L−1) · CDFclip(v),   CDFclip from histogram capped at β then redistributed

4.3  EfficientNet-B6 — deep feature extractor

Why B6: compound scaling grows depth, width and resolution together, so B6 captures fine facial texture (where morph artifacts live) far better than B0/B5 — the 62% → 66% → 90% jump in Table 2 is exactly this.

Compound scaling (depth d, width w, resolution r) d = αφ,  w = βφ,  r = γφ,   with  α·β2·γ2 ≈ 2
Convolutional block with Swish activation Xl = σ(Wl ∗ Xl−1 + bl),   σ(x) = x · sigmoid(x) = x / (1+e−x)
Global Average Pooling → fixed feature vector F = (1/N) ∑i=1N XL(i),   F ∈ ℝ2304

GAP averages each channel's spatial map to one number → a pose-robust, fixed-length descriptor, and it removes the ImageNet classifier head so the net is a pure fB6(I;θ) feature function.

4.4  Support Vector Machine — the decision

Why SVM: with only ~30k samples in a 2304-d space, a maximum-margin classifier generalises better than another deep head and resists over-fitting. It finds the single widest-margin hyperplane separating the two classes.

Labels and separating hyperplane yi ∈ {−1, +1}  (−1 real, +1 morph);   w⊤F + b = 0
Soft-margin optimisation (C balances margin vs. errors) minw,b,ξ  ½∥w∥2 + C ∑i=1N ξi
subject to  yi(w⊤Fi + b) ≥ 1 − ξi,  ξi ≥ 0
RBF kernel (non-linear boundary) & decision rule K(Fi,Fj) = exp(−γ∥Fi−Fj∥2);   ŷ = sign(∑i αiyiK(Fi,F) + b)

4.5  Evaluation & threshold — the metrics that matter

Standard metrics from the confusion matrix Acc = (TP+TN)/N,  P = TP/(TP+FP),  R = TP/(TP+FN),  F1 = 2PR/(P+R)
Morph-attack metrics (ISO/IEC 30107-3) APCER = FN/(TP+FN),  BPCER = FP/(FP+TN),  ACER = ½(APCER+BPCER)
ROC/AUC and Platt-calibrated probability AUC = ∫01 TPR d(FPR);   p(morph) = 1 / (1 + e−(A·s+B)),  s = w⊤F+b

07 · Rationale

Why each choice was made

ChoiceAlternativeWhy this one
Two-stage (features + SVM)End-to-end CNNSmall dataset → end-to-end memorises. Decoupling caps over-fitting and lets the SVM generalise.
EfficientNet-B6B0 / B5 / ResNetCompound scaling captures fine morph texture; +24 pts over B5 in practice.
Global Average PoolingFlatten denseFixed-length, pose-robust, far fewer parameters — less over-fit.
SVM (RBF)Softmax headMaximum margin on ~30k samples; smaller train/test gap than a neural head.
Seamless morphsNaive blendRemoves artifacts the model would shortcut on → learns real morphing.
Identity-disjoint splitsRandom splitPrevents identity leakage → honest, deployable accuracy.

Takeaways

The pipeline is a clean, data-efficient morph detector: a scaled CNN supplies discriminative features, a maximum-margin classifier draws a robust boundary, and the whole lifecycle is fault-tolerant and reproducible. Its measured 89.9% / 0.965 AUC is strong; its weak points — missed morphs and backbone fragility — are addressable, and the projected improvements bring accuracy toward 95% while halving the biometric error rate.
